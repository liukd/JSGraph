Formatting Graph Items in JS Graph

Overview

This chapter explains in more detail, how formats can be applied to nodes and egdes. The different fill and line styles will be introduced. Furthermore the different styles will be applied and explained in detail. Naturally, fill formats can only be applied to closed objects like rectangles, circles or closed polygons. Line formats can be applied to any GraphItem. Finally applying a shadow to a GraphItems will be explained.

Line Formatting

As a first step, line formatting will be applied. A line or the border of any other item can be formatted using different line colors, a certain line style and the line width. The color is changed using a hexadecimal color definition. This definition is preceded by a hash (#) and the Red, Green and Blue color components in hexadecimal notation ranging from 00 to FF. The line width is defined using 1/100 mm and the different line styles and given by line style constants listed in JSG.graph.attr.FormatAttributes.LineStyle. In addition you can define an edge or corner rounding for polylines. This rounding is also given in 1/100 th mm. See at the following example to find out how the parameters are applied and their effect on GraphItems.

In the next sample arrow heads are applied to a polyline. The is a set of over 20 different arrow styles defined in JSG.graph.attr.FormatAttributes.ArrowStyle, that can be applied to a line or poly line. Obviously they only show an effect if, an item is not a closed area. There the fill color is used in this case for filling the arrow head as can be seen in the sample. Here we are applying two different arrow heads and define a red fill color for the arrow heads. 


					

					
					
Fill Formats

					

						There are 3 different fill format appearances available for nodes. A solid fill, a gradient fill and a pattern fill. The type of fill to be used is defined by 
						the fill style. In the following samples you will see how the different fill styles are used. The first shows how to apply a simple fill color. In addition
						a transparency factor is set. The transparency allows, depending on the factor, to see items below the formatted item. The fill color is defined using
						the hexadecimal string as explained above. The transparency uses a percent value ranging from 0 to 100.
					

					

					

					

					
					

						In the following sample a gradient fill is defined. There are two different gradient types : the linear and the radial gradient. For the linear gradient, you 
						can set an angle, which defines the way the linear gradient is developing within the node. For radial gradients you can define the center, where the radial 
						gradient starts using a vertical and horizontal offset in percent. The fill color and the gradient color define the color range to be used. 
					

					

					

					

					
					

						The final fill style is the pattern fill. To apply a pattern you need to provide an URL to an image on any server. The pattern is displayed using the
						available Pattern styles. They are used to display a pattern either stretched, proportionally or repeated.
					

					

					

					

					
					
Shadows

					

						Now we explain, how to apply shadow settings to GraphItems. A shadow is only visible, if it is a closed shape. The shadow can be visualized using different colors.  
						The shadow depth and direction can be defined and in addition a shadow blur, which makes the shadow appear dithered.
					

					

					

					

					
					

						If the Graph has the input focus and a node is selected, you can hit Ctrl-Shift-c on the keyboard to copy the format information of an object.
						Then you can select another object and hit Ctrl-Shift-v to paste only the format information and apply it to the object.
