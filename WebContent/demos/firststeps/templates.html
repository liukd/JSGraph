Working with Templates

Overview

Templates are predefined AttributeLists including the members of the AttributeList and values that are assigned to the members or Attributes. Templates are assigned to an AttributeList to bind the AttributeList to a centralized and predefined set of settings for the Attributes. You can assign a template to muliple instance of an AttributeList. The Attributes within the AttributeList fetch their values from the Template as long as the value of the Attribute is not explicitly set. Once the Attribute receives a specific value this attribute does not retrieve its value from the template any more. But the other members of the AttributeList stay connected to the template as long as their values are not set. Whenever the template changes, the Attributes that are not explicitly set, retrieve the new values from the template.

You can use template e.g. to define styles for your GraphItems. This could imply format or font settings, but this can also be used to set predefined settings for custom AttributeLists. There are two advantages to this approach. The first advantage is that you can centralize settings, that can easily be changed afterwards. Those changes are then reflected in all instances of the Attributes (unless they were overwritten). The other advantage lies in the fact, that it saves memory, as the Attribute only needs to allocate memory, if it has a specific setting.

Using a Template

Let us look at a few examples to clarify the idea of templates. First we will create a template for formats derived from the default format template. This template is changed setting a different format and assigned to three of the four items. The first item is still bound to the default template and does not reflect any change, when the template is assigned or changed. The second and third are bound to the template and change with the template. The fourth has its own instance of the FillColor attribute and does not change this attribute, when the templates changes, but the other two.
